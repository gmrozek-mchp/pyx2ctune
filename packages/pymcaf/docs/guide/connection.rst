Connection
==========

The :class:`~pymcaf.Connection` class is the main entry point to pymcaf.
It wraps a :class:`~pymcaf.Backend` and an optional
:class:`~pymcaf.ParameterDB` to provide engineering-unit access to firmware
variables.

Creating a connection
---------------------

The simplest way to connect is via the pyX2Cscope backend:

.. code-block:: python

   from pymcaf import Connection

   conn = Connection.via_x2cscope(
       port="/dev/tty.usbmodem1",
       elf_file="firmware.elf",
       parameters_json="parameters.json",
   )

You can also construct a :class:`~pymcaf.Connection` directly with any
:class:`~pymcaf.Backend` implementation:

.. code-block:: python

   from pymcaf import Connection
   from pymcaf.backends.x2cscope import X2CScopeBackend

   backend = X2CScopeBackend("/dev/tty.usbmodem1", "firmware.elf")
   conn = Connection(backend, parameters_json="parameters.json")

The parameter database
----------------------

The ``parameters_json`` argument points to the ``parameters.json`` file
generated by motorBench.  This file contains Q-format metadata and scaling
factors for all firmware parameters, enabling pymcaf to convert between
fixed-point firmware values and real-world engineering units.

If you omit ``parameters_json``, raw variable access
(:meth:`~pymcaf.Connection.read_raw` / :meth:`~pymcaf.Connection.write_raw`)
still works, but methods that require unit conversion will raise
``RuntimeError``.

Raw variable access
-------------------

For variables that don't require Q-format conversion (integers, enums,
bitfields):

.. code-block:: python

   # Read raw value
   state = conn.read_raw("motor.state")

   # Write raw value
   conn.write_raw("motor.testing.operatingMode", 2)

Q15 conversion
--------------

For Q15-scaled variables, use the conversion methods with a fullscale
parameter key:

.. code-block:: python

   # Read in engineering units
   iq = conn.read_q15("motor.idq.q", "mcapi.fullscale.current")

   # Write in engineering units
   conn.write_q15("motor.idq.q", 0.5, "mcapi.fullscale.current")

   # Offline conversion (no hardware read/write)
   eng_value = conn.q15_to_engineering(16384, "mcapi.fullscale.current")
   counts = conn.engineering_to_q15(1.5, "mcapi.fullscale.current")

.. tip::

   In most cases, you should use the higher-level :attr:`~pymcaf.Connection.motor`
   and :attr:`~pymcaf.Connection.test_harness` interfaces instead of calling
   Q15 methods directly -- they handle variable paths and conversion
   automatically.

Disconnecting
-------------

Always call :meth:`~pymcaf.Connection.disconnect` when finished:

.. code-block:: python

   conn.disconnect()

This releases the serial port and any backend resources.
